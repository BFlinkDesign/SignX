## Codebase Patterns
- GWT RPC v7 protocol: pipe-delimited payloads, //OK[] responses, RTL data reading
- Two RPC endpoints: commandService (RunReportCommand) and ViewRPCService (report execution)
- Browser flow: ReportRPCService.lookup -> parameter dialog -> ViewRPCService POST
- HTTP replay flow: load captured payload -> send_rpc -> extract ViewToken -> paginate via getData
- Auth: ERP login -> SSO -> JSESSIONID + authToken + clientId
- Field discovery: 3-tier fallback (hardcoded -> manifest.json -> auto-discovery from GWT response)
